RViz UI

  • The RViz UI automatically pops up when the robot requires assistance
  • Remember to speak out loud while using the UI
  • You MAY:
    • Change the RViz camera view with your mouse:
      • left-click: tilt/rotate
      • middle-click/right-click: zoom
      • shift+left-click: pan
    • Select or unselect any of the displays available in the Displays panel (left).
    • Use the 2D Pose Estimate control in the top bar to provide the robot with a pose estimate.
    • Resize the window and/or panels as you see fit
  • You MAY NOT:
    • Add additional displays to the UI
    • Change the options for the displays that are available; even if there is an error in them
    • Use any other control that is not explicitly allowed above

Note: Large data streams (PointCloud2, PlanningScene) may be delayed because of the network, please be patient.

Perception Example
  • RobotModel shows the robot
  • LaserScan shows the laser scan from the robot as a series of points (shown mostly in blue in the image)
  • PointCloud2 shows a coloured point cloud from the robot's RGBD camera
  • Image shows an RGB image of the scene visible in the point cloud
  • Map shows the map the robot is operating within, shown as an occupancy grid
Navigation Example
  • TF shows all the reference frames in the map that the robot uses to know what-is-where. Some of these frames correspond to the locations indicated in the tasks map
  • Navigation/Pose shows the last navigation goal pose that was sent to the robot as a large red arrow (may occasionally indicate an error).
Motion Planning Example

The static octomap that is used to seed the planning scene

The planning scene

  • StaticOctomap shows a down-sampled 3D map of the environment that is used to seed the PlanningScene
  • PlanningScene shows an obstacle map constructed from StaticOctomap and PointCloud2, which is used for arm motion planning
    • The display includes a RobotModel of its own
    • Simple geometric shapes in purple (like the cuboid in the image) are additional obstacles that can be programmatically added to the scene
  • Contacts shows potential collisions (as red spheres) that were explored (but not executed) during arm motion planning
  • Trajectory shows planned arm motions during an arm motion planning command
Grasping Example

An example segmentation of the objects on a table

Calculated grasp pose on segmented object i: 1

  • SegmentedObjects show individual objects detected by the robot on a table-top; the data is only refreshed after running the segment action. In the task, the resulting segmented point clouds are used to recognize objects of interest and calculate grasps on them.
  • Grasping/Pose shows the grasp (approach) pose calculated by the robot on a particular object of interest. The calculation is usually done once the object of interest is recognized from the segmented point clouds
